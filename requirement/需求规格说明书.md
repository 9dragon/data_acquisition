## 1. 项目概述
### 1.1 项目背景
随着工业4.0和数字化转型的深入推进，制造企业需要将工厂现场的各类生产设备、PLC、CNC、机器人、传感器等设备的数据采集到数字化平台，以实现生产过程的可视化、分析和优化。然而，数据采集项目涉及多设备、多协议、多阶段、多人员的复杂协作，缺乏有效的管理系统容易导致进度失控、问题积压、沟通不畅等问题。

本系统旨在为工业数据采集项目提供一套完整的数字化管理平台，覆盖项目从立项到验收的全生命周期，帮助项目团队高效协同，确保项目按时按质交付。

### 1.2 建设目标
#### 1.2.1 核心目标
1. **项目全生命周期管理**
    - 实现项目从售前调研到最终验收的六阶段全过程管理
    - 提供项目计划的制定、执行、监控和调整能力
    - 支持多项目并行管理和资源协调
2. **设备采集进度可视化**
    - 实时跟踪每台设备的数据采集进度
    - 支持点位级别的进度监控（总点位数 vs 已采集点位数）
    - 提供多维度进度统计和预警
3. **问题快速响应与处理**
    - 建立完整的问题跟踪和协同处理机制
    - 支持问题的快速上报、分配、处理、验证流程
    - 提供问题统计和分析能力，持续改进实施质量
4. **知识沉淀与文档管理**
    - 建立项目文档的集中管理和分类体系
    - 支持文档的版本控制和标签管理
    - 促进实施经验和知识的复用

#### 1.2.2 业务价值
+ **提升项目交付效率**: 通过标准化流程和工具支持，提高项目执行效率
+ **降低沟通成本**: 统一的信息平台减少跨部门沟通误解和信息遗漏
+ **提高客户满意度**: 透明的进度展示和及时的问题响应提升客户信任度
+ **积累组织能力**: 项目经验和知识的沉淀提升团队整体能力

### 1.3 适用范围
#### 1.3.1 业务场景
本系统适用于以下业务场景：

+ **工业设备数据采集项目**: 包括PLC、CNC、机器人、传感器、仪表等设备的数据采集
+ **工业物联网实施项目**: 涉及多协议（OPC UA、Modbus、MQTT、HTTP等）的设备接入
+ **数字化工厂建设项目**: 作为数字化工厂平台的设备数据采集实施管理工具

#### 1.3.2 用户角色
| 角色 | 职责描述 |
| --- | --- |
| **项目经理** | 负责项目整体规划和进度管理，协调资源，控制项目风险 |
| **实施工程师** | 负责设备配置、点位采集、测试验证等具体实施工作 |
| **技术负责人** | 负责技术方案设计、技术问题攻关、技术审核 |
| **系统管理员** | 负责用户管理、权限配置、系统维护 |


### 1.4 系统边界
#### 1.4.1 系统范围
**包含功能**：

+ 项目管理和计划管理
+ 设备信息和采集进度管理
+ 任务分配和进度填报
+ 问题跟踪和处理协同
+ 文档管理和知识沉淀
+ 用户权限和系统配置

**不包含功能**：

+ 实际的设备数据采集功能（由专门的采集平台完成）
+ 数据存储和分析（对接上游数据平台）
+ 项目财务和成本管理
+ 采购和供应链管理

#### 1.4.2 系统集成
系统需要与以下外部系统集成：

+ **设备采集平台**: 获取设备采集进度和点位数据
+ **统一身份认证**: 实现单点登录和用户同步
+ **消息通知平台**: 发送待办提醒和通知消息

### 2.2 系统管理模块
#### 2.2.1 用户管理
**功能描述**: 提供用户账号的管理功能。

**详细需求**:

1. **用户列表**
    - 显示用户名、姓名、邮箱、手机号
    - 显示所属角色、状态
    - 显示创建时间、最后登录时间
    - 支持按用户名、姓名搜索
    - 支持按角色、状态筛选
2. **用户操作**
    - 新增用户：填写用户名、密码、姓名、邮箱、手机号、角色
    - 编辑用户：修改姓名、邮箱、手机号、角色
    - 重置密码：管理员可重置用户密码
    - 启用/禁用用户
    - 删除用户：仅可删除无任何关联数据的用户
3. **用户信息**
    - 用户名：必填，唯一，3-20个字符
    - 密码：必填，8-20个字符，必须包含字母和数字
    - 姓名：必填
    - 邮箱：必填，格式校验，唯一
    - 手机号：选填，格式校验
    - 角色：必选，支持多选
    - 状态：启用/禁用

#### 2.2.2 个人资料管理
**功能描述**: 提供用户个人信息的查看和修改功能。

**详细需求**:

1. **个人信息查看**
    - 显示用户名、姓名、邮箱、手机号
    - 显示所属角色
    - 显示头像
    - 显示最后登录时间
2. **个人信息修改**
    - 修改姓名
    - 修改邮箱
    - 修改手机号

3. **安全设置**
    - 修改密码：需要输入原密码
    - 密码强度提示

4. **偏好设置**
    - 界面语言：中文/英文
    - 主题设置：浅色/深色

**界面要求**:

+ 采用表单布局，分组展示
+ 头像支持点击上传和预览
+ 敏感信息（如密码）采用隐藏显示

### 2.3 设备管理模块
#### 2.3.1 设备列表
**功能描述**: 提供设备的查询、筛选和快捷操作。

**详细需求**:

1. **列表展示**
    - 显示设备编码、名称、所属项目
    - 显示设备类型、采集方式
    - 显示设备状态（未开始/进行中/已完成/异常）
    - 显示点位进度（总点位数/已采集点位数）
    - 显示总进度（进度条+百分比）
    - 显示负责人
    - 支持按多列排序
2. **查询筛选**
    - 支持按设备编码/名称模糊搜索
    - 支持按所属项目筛选
    - 支持按设备类型筛选（PLC/CNC/Robot/Sensor/Instrument/其他）
    - 支持按采集方式筛选（OPC UA/Modbus TCP/Modbus RTU/MQTT/HTTP/其他）
    - 支持按设备状态筛选
    - 支持按负责人筛选
    - 支持多条件组合筛选
3. **快捷操作**
    - 点击设备行进入设备详情页
    - 操作列包含"查看"、"编辑"、"删除"操作按钮
    - 支持新建设备
    - 支持批量操作（批量分配、批量更新状态、批量导出）

#### 2.3.2 设备详情
**功能描述**: 展示设备的完整信息和采集进度。

**详细需求**:

1. **基本信息区**
    - 设备编码、名称
    - 所属项目（可点击进入项目详情）
    - 设备类型、采集方式
    - 设备描述、技术参数
    - 设备状态、负责人
2. **连接信息区**
    - IP地址、端口
    - 协议参数
    - 连接状态（在线/离线）
    - 最后连接时间
3. **采集进度区**
    - 总点位数、已采集点位数、未采集点位数
    - 点位进度条
    - 总进度
    - 各阶段进度（准备/施工/配置/核对）
    - 最后更新时间、更新人
4. **关联信息区**
    - 关联项目：显示项目名称、阶段、进度
    - 关联问题：显示该设备的待处理问题
    - 关联文档：显示该设备的技术文档
5. **操作区**
    - 编辑设备信息
    - 更新采集进度
    - 创建问题
    - 上传文档
    - 删除设备

#### 2.3.3 设备创建/编辑
**功能描述**: 提供设备信息的创建和编辑功能。

**详细需求**:

1. **基本信息**
    - 设备编码：必填，唯一性校验
    - 设备名称：必填
    - 所属项目：必选
    - 设备类型：必选（PLC/CNC/Robot/Sensor/Instrument/其他）
    - 采集方式：必选（OPC UA/Modbus TCP/Modbus RTU/MQTT/HTTP/其他）
    - 设备描述：选填，支持富文本
2. **连接信息**
    - IP地址：选填，IP格式校验
    - 端口：选填，数字范围校验（1-65535）
    - 协议参数：根据采集方式动态显示相应参数字段
3. **采集信息**
    - 总点位数：必填，正整数
    - 负责人：必选
4. **表单验证**
    - 必填项检查
    - 编码唯一性检查
    - IP地址格式检查
    - 端口范围检查

#### 2.3.4 设备类型管理
**功能描述**: 提供设备类型的分类管理，支持按项目、工序进行筛选和联动。

**详细需求**:

1. **类型列表**
    - 显示所属项目、所属工序、类型编码、名称、描述
    - 显示该类型的设备数量
    - 支持排序、搜索

    - 切换项目时自动清空工序筛选
2. **类型管理**

#### 2.6.6 问题工作流
**功能描述**: 定义问题的生命周期和状态流转规则。

**详细需求**:

1. **标准流程**
    - 待处理 → 已分配 → 进行中 → 已解决 → 已关闭
    - 待处理 → 已关闭（非问题、重复等问题）
2. **状态说明**
    - **待处理**: 问题刚创建，尚未分配负责人
    - **已分配**: 已分配负责人，但尚未开始处理
    - **进行中**: 负责人正在处理中
    - **已解决**: 问题已修复，待验证
    - **已关闭**: 问题已验证通过或无需处理
3. **状态权限**
    - 所有人可创建问题（状态为"待处理"）
    - 项目经理可分配负责人（状态变为"已分配"）
    - 负责人可开始处理（状态变为"进行中"）
    - 负责人可标记为已解决（状态变为"已解决"）
    - 报告人或项目经理可验证关闭（状态变为"已关闭"）
4. **自动流转**
    - 问题创建后自动通知负责人
    - 分配负责人后通知相关人员
    - 状态变更时通知报告人和抄送人
    - 问题即将逾期时发送预警通知

**功能描述**: 提供任务的创建、编辑、分配、跟踪功能。

**详细需求**:

1. **任务创建**
    - 任务名称：必填
    - 任务描述：选填，支持富文本
    - 所属项目：必选
    - 所属阶段：必选（准备/施工/配置/核对）
    - 任务状态：默认"未开始"，可选（未开始/进行中/已完成/已暂停）
    - 优先级：必选（低/中/高/紧急）
    - 负责人：必选，支持多选
    - 计划开始时间：必选
    - 计划结束时间：必选，必须晚于开始时间
    - 预计工时：选填，单位小时
    - 完成进度：0-100%，默认0
    - 前置任务：可选，支持多选，建立任务依赖关系
    - 任务标签：可选，支持多选
2. **任务编辑**
    - 支持编辑所有字段
    - 记录修改历史
3. **任务操作**
    - 开始任务：状态变为"进行中"
    - 完成任务：状态变为"已完成"，进度自动变为100%
    - 暂停任务：状态变为"已暂停"
    - 删除任务：仅可删除未开始的任务
    - 分配任务：重新分配负责人
4. **任务依赖**
    - 支持设置前置任务（完成-开始、开始-开始、完成-完成、开始-完成关系）
    - 前置任务未完成时，后续任务不能开始
    - 依赖关系可视化展示

#### 2.7.4 进度填报
**功能描述**: 提供设备各阶段进度的填报功能。

**详细需求**:

1. **快速填报**
    - 选择设备：显示设备基本信息和当前进度
    - 选择阶段：准备/施工/配置/核对
    - 输入进度：0-100%的数字输入或滑块选择
    - 填报说明：选填，文字说明
    - 支持批量选择多个设备同时填报
2. **填报规则**
    - 进度必须为0-100的数字
    - 后续阶段的进度不能超过前一阶段（如施工阶段进度不能超过准备阶段）
    - 填报后立即生效，无需审核
    - 记录填报历史（时间、填报人、进度值、说明）
3. **进度计算**
    - 阶段进度 = 该阶段下所有任务的平均进度
    - 设备总进度 = Σ(各阶段进度 × 阶段权重) / 100
    - 项目总进度 = 所有设备的平均进度
4. **进度历史**
    - 显示每个设备的进度填报历史
    - 支持按时间、阶段筛选
    - 支持导出进度报告

4. **时间维度**

    - 预计完成时间预测
    - 延期预警（当前进度落后于计划）
5. **可视化**
    - 进度条、饼图、柱状图、折线图
    - 支持导出为图片或PDF
    - 支持定时发送报告邮件

### 3.2 业务规则
#### 3.2.1 项目相关规则
**规则1：项目阶段流转**

+ 项目必须按阶段顺序进行：售前调研 → 准备阶段 → 施工阶段 → 配置阶段 → 核对阶段 → 验收阶段
+ 对于按设备推进的阶段，必须上一阶段完成后才能进入下一阶段
+ 一个项目允许同时处于多个阶段，以最新的阶段判定项目当前阶段
+ 项目经理可以手动调整项目阶段

**规则2：项目进度计算**

```plain
总体进度 = Σ(各阶段进度 × 阶段权重) / 100
```

+ 阶段进度 = 该阶段下所有设备的平均进度
+ 权重总和必须等于100%
+ 售前调研和验收阶段不计入实施进度

**规则3：项目状态管理**

+ 项目创建时状态为"未开始"
+ 项目启动后状态变为"进行中"
+ 所有设备完成后状态可变为"已完成"
+ 项目暂停时状态变为"已暂停"
+ 项目取消时状态变为"已取消"

**规则4：项目删除**

+ 已开始的项目不能删除
+ 已关联设备、问题、文档的项目不能删除

#### 3.2.2 设备相关规则
**规则1：设备进度计算**

```plain
总进度 = Σ(各阶段进度 × 阶段权重) / 100
```

+ 后续阶段的进度不能超过前一阶段

**规则2：设备删除**

+ 已开始采集的设备不能删除
+ 已关联问题的设备不能删除

#### 3.2.3 进度填报规则
**规则1：进度填报限制**

+ 进度值自动计算，不允许人为手动修改

#### 3.2.4 问题管理规则
**规则1：问题状态流转**

+ 待处理 → 已分配（分配负责人）
+ 已分配 → 进行中（负责人开始处理）
+ 进行中 → 已解决（负责人提交解决方案）
+ 已解决 → 已关闭（报告人验证通过）
+ 已解决 → 进行中（验证不通过，重新处理）
+ 待处理 → 已关闭（非问题、重复等问题）

**规则2：问题分配**

+ 问题创建后必须分配负责人
+ 负责人必须为项目团队成员
+ 支持重新分配负责人

**规则3：问题通知**

+ 问题创建后通知负责人
+ 分配负责人后通知相关人员
+ 状态变更时通知报告人和抄送人
+ 问题逾期时发送预警

**规则4：问题关闭**

+ 仅报告人和项目经理可关闭问题
+ 关闭时必须填写关闭原因
+ 关闭后不可重新打开

#### 3.2.5 任务管理规则
**规则1：任务依赖**

+ 任务可设置前置依赖任务
+ 前置任务未完成，后续任务不能开始
+ 支持四种依赖关系：完成-开始(FS)、开始-开始(SS)、完成-完成(FF)、开始-完成(SF)

**规则2：任务完成**

+ 任务完成后，进度自动变为100%
+ 任务完成后，状态变为"已完成"
+ 记录实际完成时间

**规则3：任务删除**

+ 已开始的任务不能删除
+ 有后续依赖的任务不能删除

#### 3.2.6 文档管理规则
**规则1：文档上传**

+ 支持单个和批量上传
+ 单个文件大小不超过100MB
+ 文件名不可重复（同一项目下）

**规则2：文档版本**

+ 上传同名文件时自动创建新版本
+ 保留历史版本
+ 显示当前版本号

**规则3：文档权限**

+ 项目成员可查看项目文档
+ 项目成员可上传文档
+ 仅上传人和管理员可删除文档
+ 支持设置文档访问权限

#### 3.2.7 用户权限规则
**规则1：角色分配**

+ 用户必须至少分配一个角色
+ 角色权限可叠加
+ 系统管理员角色不可删除

**规则2：权限继承**

+ 项目经理拥有项目下所有资源的完全控制权
+ 设备负责人仅拥有该设备的操作权限
+ 只读用户仅拥有查看权限

**规则3：数据隔离**

+ 用户只能看到有权限的项目和设备
+ 问题可设置抄送，抄送人可查看但不可操作
+ 文档可设置可见人员

#### 3.2.8 设备调研管理规则
**规则1：调研创建方式**

+ 支持从零创建：填写完整的设备基本信息和调研信息
+ 支持基于设备创建：自动填充设备的部分信息（编号、名称、项目、车间等）
+ 从零创建的调研，设备信息可编辑
+ 基于设备创建的调研，设备信息从设备带出，不可编辑
+ 一个设备只能有一条调研记录

**规则2：调研进度计算**

```plain
调研进度 = (基础信息完成度 + 控制器信息完成度 + 采集信息完成度) / 3
```

+ 每个模块完成度：0%（未完成）或 100%（已完成）
+ 基础信息保存后，基础信息完成度 = 100%
+ 控制器信息保存后，控制器信息完成度 = 100%
+ 采集信息保存后，采集信息完成度 = 100%
+ 总体进度为整数，范围0-100

**规则3：调研状态流转**

+ **未开始**：三个模块都未完成（进度=0%）
+ **进行中**：至少一个模块已完成，但未全部完成（0% < 进度 < 100%）
+ **已完成**：三个模块都已完成（进度=100%）
+ 保存任一模块信息后，状态自动从"未开始"变为"进行中"
+ 所有模块完成后，状态自动变为"已完成"

**规则4：多媒体文件管理**

+ 支持图片和视频上传
+ 图片类型：控制器照片、触摸屏照片、控制柜照片
+ 视频类型：控制器视频、触摸屏视频、控制柜视频
+ 单个图片文件大小限制：默认10MB
+ 单个视频文件大小限制：默认50MB
+ 图片数量限制：每个类型最多10张
+ 视频数量限制：每个类型最多5个
+ 支持文件预览（图片预览、视频播放）
+ 支持文件删除

**规则5：数据导入导出**

+ **导入规则**：
    - 支持Excel文件导入（.xlsx、.xls）
    - 导入时验证设备编号存在性
    - 导入时验证数据完整性和格式正确性
    - 显示导入结果（成功数、失败数、错误详情）
    - 设备编号不存在的记录导入失败
+ **导出规则**：
    - 支持导出选中的调研记录
    - 导出格式为Excel文件
    - 包含所有调研信息（基础信息、控制器信息、采集信息）
    - 不包含多媒体附件（附件URL可作为链接导出）

**规则6：调研信息保存**

+ 每个模块独立保存
+ 保存时验证必填项和数据格式
+ 保存成功后标记该模块为已完成
+ 自动重新计算总体进度
+ 保存失败时显示错误信息，不更新完成状态
+ 支持多次编辑和保存

**规则7：调研权限控制**

+ 项目经理可查看和编辑项目下所有设备的调研记录
+ 设备负责人可查看和编辑所负责设备的调研记录
+ 调研人员可查看和编辑自己创建的调研记录
+ 查看模式下，所有信息只读，不可编辑
+ 编辑模式下，可修改调研信息
+ 删除调研记录需确认，删除后不可恢复

#### 3.2.9 工序管理规则
**规则1：工序层级关系**

+ 工序从属于项目
+ 一个项目可以包含多个工序
+ 工序之间可设置排序顺序（sortOrder）
+ 工序按排序顺序显示

**规则2：设备类型与工序关联**

+ 设备类型可以关联工序（可选）
+ 关联工序后，设备类型按工序分组显示
+ 工序筛选显示该工序下的所有设备类型
+ 选择项目后，设备类型管理页面的工序筛选只显示该项目的工序
+ 切换项目时，已选工序自动清空

## 5. 非功能性需求
### 5.1 性能要求
#### 5.1.1 响应时间
**页面加载**:

+ 首屏加载时间 ≤ 2秒（正常网络环境）
+ 后续页面切换时间 ≤ 1秒
+ 表格数据加载时间 ≤ 1秒（100条以内）

**操作响应**:

+ 按钮点击响应时间 ≤ 200ms
+ 表单提交响应时间 ≤ 500ms
+ 搜索结果返回时间 ≤ 1秒

### 5.2 安全要求
#### 5.2.1 身份认证
**认证方式**:

+ 用户名密码登录
+ 支持第三方登录（LDAP、OAuth等）
+ 支持单点登录（SSO）

**密码策略**:

+ 密码长度8-20个字符
+ 必须包含字母和数字
+ 支持密码强度提示
+ 密码加密存储（BCrypt等）

**会话管理**:

+ 会话超时时间：30分钟（可配置）
+ 支持记住登录状态（7天）
+ 异常登录检测和提醒

#### 5.2.2 权限控制
**权限模型**:

+ 基于角色的访问控制（RBAC）
+ 支持权限继承和组合
+ 支持数据级权限控制

**权限验证**:

+ 前端路由权限验证
+ 后端接口权限验证
+ 数据级权限过滤

#### 5.2.3 数据安全
**数据传输**:

+ 全站HTTPS加密
+ 敏感数据加密传输
+ 防止SQL注入、XSS攻击

**数据存储**:

+ 密码等敏感信息加密存储
+ 定期数据备份
+ 敏感操作日志记录

**数据隔离**:

+ 多租户数据隔离（如果支持）
+ 用户只能访问有权限的数据

#### 5.2.4 审计日志
**日志内容**:

+ 用户登录/登出
+ 关键操作（创建、编辑、删除）
+ 权限变更
+ 数据导出

**日志要求**:

+ 记录操作人、操作时间、操作内容、操作结果
+ 日志不可篡改
+ 支持日志查询和导出
+ 日志保留时间 ≥ 180天
